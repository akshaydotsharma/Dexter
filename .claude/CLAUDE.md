# Personal Dashboard

A solo-built productivity stack: web client (React 19 + Vite + Tailwind 4), Node.js/Express backend on PostgreSQL, and a native iOS companion app in SwiftUI. Uses Anthropic Claude for the chat / AI draft features. The repo is a monorepo: `client/`, `server/`, `mobile/`.

## Tech stack at a glance

- **Frontend (web)**: React 19, Vite 7, Tailwind CSS 4, MUI date pickers, axios, lucide-react. Dev server on :5173.
- **Frontend (iOS)**: SwiftUI, `URLSession.AsyncBytes` for SSE streaming, `xcodegen` for project generation, free personal-team signing for development builds (7-day expiry).
- **Backend**: Node.js 20+, Express 5, PostgreSQL via `pg`. Default API port :3000. Vercel AI SDK + `@ai-sdk/anthropic` for chat. Express rate limiting + helmet for security.
- **Tests**: `node --test` against a dedicated `personal_dashboard_test` Postgres database. Run with `npm test` from `server/`.

## SSE wire format (chat)

`/api/ai/parse/stream` emits a typewriter-style SSE stream that BOTH the iOS app and the web client speak. Do not change the frame format without updating both clients.

```
event: drafts   data: {"drafts": [...]}            (once, before text)
event: text     data: {"chunk": "..."}             (zero or more)
event: done     data: {"followUpQuestion"?, "errors": [...]}
event: error    data: {"message": "..."}
```

A real `streamText`-based implementation (`{type:"delta"/"draft"/"done"}` frames) was prototyped on the dropped `refactor/v2-backend` `chatStream.js`. To revive it, restore that file AND update `mobile/PersonalDashboard/Services/AIStreamingService.swift` + `client/src/hooks/useChat.js` together — the wire format is breaking.

## Project-specific rules

- Every secret-bearing file (`server/.env`, `client/.env`) is gitignored. Templates live in `*.env.example`. Never commit a real key.
- `mobile/PersonalDashboard.xcodeproj/` is gitignored — regenerate via `xcodegen generate` from `mobile/project.yml`.
- The iOS free personal-team profile expires every 7 days. Re-run `bash mobile/ota/ship.sh` to ship a fresh build to a phone over a Cloudflare quick tunnel. Cloudflare quick tunnels are ephemeral; `AppConfig.swift`'s API URL becomes stale when the tunnel dies. Override at archive time with `API_URL` env var when the device is on a different host than localhost.
- Two server processes can collide: the dev server (`npm run dev` -> nodemon) and a stale `node server/index.js` started directly. If `/api/ai/parse/stream` 404s, check `lsof -nP -iTCP:3000 -sTCP:LISTEN` for a stale pid running pre-streaming code.
- Migration philosophy: `schema.sql` is the canonical brand-new install; `migration.sql` is the cumulative idempotent diff applied to existing DBs. Both run on server boot. New columns must be additive and idempotent.
- Drag-to-reorder uses gap-1000 sequential `position INTEGER` columns. The `computeNewPosition` helper renumbers a scope when gaps collapse to <= 1.

## Dev workflow

```bash
# from repo root, fresh clone
cp server/.env.example server/.env   # set ANTHROPIC_API_KEY, DATABASE_URL
cp client/.env.example client/.env   # optional
npm run install:all                  # installs root + client + server
npm start                            # runs server + client together (concurrently)
```

Tests:
```bash
cd server
createdb personal_dashboard_test     # one-time
npm test                             # node --test against the test DB
```

iOS:
```bash
cd mobile
xcodegen generate                    # regenerate .xcodeproj from project.yml
open PersonalDashboard.xcodeproj     # for simulator / device runs
bash ota/ship.sh                     # OTA install on a paired iPhone
```

## Deployment

- Backend: Railway (`railway.json`).
- Frontend: served as static files from `client/dist` in production by the same Express process.
- iOS: free dev builds shipped via OTA + Cloudflare quick tunnels. No App Store distribution today.
